> Aufklärung ist der Ausgang des Menschen aus seiner selbst verschuldeten Unmündigkeit.

Immanuel Kant, 1783.

![](/static/images/Kant_gemaelde_3.jpg)


So sah dies Unterschrift dieses Mannes aus:

![](/static/images/Immanuel_Kant_signature.svg)

Unterschriften -- bis heute sind sie wesentliches Zeichen unserer Mündigkeit.

Sie sind sogar ein wesentlicher Bestandteil unserer Identität. **Im digitalen Zeitalter
sind deshalb
digitale Unterschriften der wichtigste Bestandteil einer digitalen Identität**.

Auf Blockchains wird
jede Transaktion digital signiert, und damit auch jede Finanztransaktion.
Wer ein Krypto-Wallet benutzt, kann diese Fuktionalität nutzen, um
beliebige Dokumente signieren. Wenigen Zeilen Javascript zeigen,
wie das aussehen könnte:

```ts
import { ethers, Signer } from 'ethers'

  async function signerDemo(signer, message) {
    if (
      (await signer.getAddress()) ===
      ethers.verifyMessage(message, await signer.signMessage(message))
    ) {
      [...]
    }
  }

```

Dieser Code könnte so ähnlich in einem Web-Browser laufen.
Unter jeden (kurzen!) Text kann auf diese Weise eine
digitale Unterschrift gesetzt werden. Bei grösseren Dokumenten wird ein Hashing-Verfahren [^1]
eingesetzt und anschliessend der Hash signiert. Ebenso kann man Bilder signieren, Musik, Filme,
jede beliebige Datenmenge.

Wo Dokumente signiert werden können, da ist es nicht mehr weit bis zu Regelwerken aller Art --  Konstitutionen,
Verfassungen, multinationale Verträge. Wer unterschreiben kann, kann wählen.
Kaufen, verkaufen, leihen, vererben, erben, sich irgendwie verpflichten. Fast jedes wichtige Recht, das wir heutzutage
als Staatsbürger ausüben können, ist an eine Unterschrift gekoppelt.

Unser Leben beginnt mit der Unterschrift unter einen Geburtsschein. Kindergarten, Schule, Universität -- fast
jeder wichtige Schritt, den wir im Leben tun, wird mit einer Unterschrift besiegelt. Ob wir heiraten
oder uns scheiden lassen, ob wir ein Haus oder ein Auto kaufen, ob wir eine Arbeitstelle antreten
oder einen Kredit aufnehmen -- immer beginnt alles mit unserer Unterschrift. Unser Leben endet mit dem
Totenschein. Über Unterschriften
regelt der Staat von der Wiege bis zur Bahre einfach alles.

![](/static/images/digital_signatures.png)

Auch Geld, insbesondere Papiergeld ist einem signierten Schuldschein nicht unähnlich:
Ein Staat bescheinigt darauf durch seine Unterschrift, dass er ein Papier als rechtmässiges
Zahlungsmittel anerkennt. Kryptowährungen benötigen keine Signatur, behalten aber das Konzept
*Geld* bei und stehen deshalb zu Fiat-Währungen in einem Konkurrenzverhältnis.

Wir ziehen jetzt eine Verbindungslinie vom Ethereum-Account eines menschlichen Individuums,
über dessen digitale Signatur hin zu seiner digitalen Identität:

![](/static/images/SignedDocuments.png)

Die Graphik zeigt ein Potential, das zwar auf technischer Ebene schon realisiert ist, bis jetzt aber
noch weit hinter dem zurück bleibt, was eigentlich möglich ist.

Wie schon erwähnt, ermöglicht der Besitz eines "Crypto-Wallets" prinzipiell jedermann, beliebige
Dokumente zu signieren bzw. zu unterschreiben. Wie das geht, habe ich bereits skizziert.

**Fallbeispiel**: Jemand erstellt ein digitales Bild seines Passes und signiert es dann mit seinem
Krypto-Wallet.
Damit kann überprüft werden, ob der Eigentümer des Passes identisch mit dem Eigentümer des "Krypto-Wallets"
ist. Das geschieht so wie in dem obigen Coding-Beispiel demonstriert.

Der eingescannte Pass darf nicht unkontrolliert über das Internet
verbreitet wird, da sonst Missbrauch möglich wird. Der Pass darf nur für den Eigentümer zugreifbar sein,
genau wie das bei einem physischen
Pass auch der Fall ist. Der signierte **Hash** des Passes dagegen kann an beliebiger Stelle veröffentlicht
werden, denn aus ihm lassen sich die Passdaten nicht wieder herstellen.

So kann sich ein Individum kann sich
gegenüber
einem vertrauenswürdigen Institution / einem vertrauenswürdigen Unternehmen identifzieren. Dazu muss er
nur noch den eingescanten Pass vorweisen. Dieser kann dann über die Signatur
**genau einem Ethereum-Account** zugeordnet werden.

**So ist mit geringem Aufwand eine digitale Identität entstanden.**

[^1]: Ein digitales Hash-Verfahren ist ein Algorithmus,
der Daten jeder Größe (wie Text, Bilder oder jede andere Datei) nimmt
und daraus einen festen Größe String oder Zahlenwert, genannt Hashwert, erzeugt.
Dieser Prozess ist einseitig:
Während es einfach ist, aus den ursprünglichen Daten den Hashwert zu
berechnen, ist es praktisch unmöglich,
aus dem Hashwert die ursprünglichen Daten zu rekonstruieren.
Das Besondere an einem Hashwert ist, dass er einzigartig für die spezifischen Eingabedaten ist.
Selbst eine kleine Änderung in den Daten führt zu einem völlig anderen Hashwert.
Diese Eigenschaft macht Hash-Verfahren nützlich für die Überprüfung
der Integrität von Daten, da man leicht erkennen kann,
ob sich die Daten verändert haben, indem man den Hashwert überprüft.
